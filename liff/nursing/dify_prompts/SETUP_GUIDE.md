# Difyワークフロー LLMノード設定ガイド
## はなまるAI 訪問看護 — SOAP→報告書生成ワークフロー

---

## ファイル構成

```
dify_prompts/
├── llm_node_a_doctor_report.txt      ← LLMノードA：医師向け報告書
├── llm_node_b_caremanager_report.txt ← LLMノードB：ケアマネ向け報告書
└── SETUP_GUIDE.md                    ← このファイル
```

---

## STARTノードの変数定義（確認用）

| 変数名           | 型       | 必須 | 説明                                     |
|-----------------|---------|------|------------------------------------------|
| patient_info    | string  | 必須 | 患者基本情報（氏名・年齢・性別・診断名・介護度） |
| soap_own        | string  | 必須 | 担当看護師自身のSOAP記録（複数件・テキスト結合）  |
| soap_other      | string  | 任意 | 他スタッフのSOAP記録（空文字列可）              |
| soap_extra      | string  | 任意 | 追加で貼り付けた記録（空文字列可）               |
| doc_instruction | string  | 推奨 | 医師の訪問看護指示書                          |
| doc_care_plan   | string  | 任意 | ケアマネの居宅サービス計画書（空文字列可）        |

## Dify LLMノード設定手順

### LLMノードA（医師向け）

1. ワークフローキャンバスで「LLM」ノードを追加
2. ノード名を「医師向け訪問看護報告書生成」に設定
3. モデル設定：推奨 `gpt-4o` または `claude-3-5-sonnet`
   - 温度（Temperature）: 0.3（創作を抑え事実ベースの出力を促す）
   - 最大トークン: 1500
4. **SYSTEMプロンプト欄**に `llm_node_a_doctor_report.txt` の
   「■ SYSTEM プロンプト」セクションの内容を貼り付ける
5. **USERプロンプト欄**に同ファイルの「■ USER プロンプト」セクションを貼り付ける
6. 変数参照が `{{patient_info}}` のような二重波括弧で設定されていることを確認

### LLMノードB（ケアマネ向け）

1. ワークフローキャンバスで「LLM」ノードを追加（ノードAと並列配置）
2. ノード名を「ケアマネ向け訪問看護報告書生成」に設定
3. モデル設定：LLMノードAと同じ設定を推奨
4. **SYSTEMプロンプト欄**に `llm_node_b_caremanager_report.txt` の
   「■ SYSTEM プロンプト」セクションの内容を貼り付ける
5. **USERプロンプト欄**に同ファイルの「■ USER プロンプト」セクションを貼り付ける

## チューニングのポイント

### 緊急性の感度調整
- 過検出が多い場合：SYSTEMプロンプトの緊急性ルールに
  「明らかな数値異常（例：収縮期血圧180以上、SpO₂90%未満など）がある場合のみ」
  という閾値を追加する。
- 見落としが心配な場合：逆に閾値を下げるか、Pセクションの【緊急】条件を広げる。

### 文字数のコントロール
- 長すぎる場合：SYSTEMプロンプトの「全体の目安文字数は500〜900字」の上限を下げる。
- 詳細が足りない場合：上限を1200字程度に変更する。

### 専門用語の変換（ノードB）
- 現場でよく出る専門用語は SYSTEMプロンプトの例示リストに追記すると精度向上。
  例：「「誤嚥性肺炎」→「食べ物が肺に入ることで起きる肺炎」」

---

## 注意事項

- 記録にない情報をAIが補完・推測しないよう、USERプロンプト末尾で明示的に禁止しています。
- 患者情報は個人情報のため、テスト時は必ず匿名化したダミーデータを使用してください。
- Difyの変数参照は `{{変数名}}` 形式（二重波括弧）です。シングル波括弧 `{変数名}` では機能しません。
